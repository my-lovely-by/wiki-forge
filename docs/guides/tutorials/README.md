# Tutorials

Learning-oriented user docs, per [Diátaxis](https://diataxis.fr/). A
tutorial takes a beginner from nothing to a small complete success —
step-by-step, with concrete commands and predictable outputs. The
reader follows along and produces a working result by the end.

## Index

- **[Tutorial 1 — Create your first vault](tutorial-1-first-vault.md)**
  — go from a fresh install to a working vault on disk. Uses the
  `personal` recipe (the smallest). Recipe-agnostic in spirit: every
  step transfers to `family` and `work-os` with the recipe name
  swapped.
- **[Tutorial 2 — A work-os walkthrough](tutorial-2-work-os-walkthrough.md)**
  — the same flow with the `work-os` recipe, walking
  `stakeholder-update` → `stakeholder-map-refresh` end to end.
- **Tutorial 3 — Family walkthrough.** Coming in the next milestone.
  The family recipe focuses on household OS (meals, medical, trips,
  vendors). Until it ships, `examples/family-mini/` in the repo is a
  populated reference you can browse.

## Other Diátaxis buckets

- **[How-to guides](../how-to/)** — task-oriented; the reader has a
  problem and wants a recipe.
- **[Reference](../reference/)** — information-oriented; the reader
  is looking something up.
- **[Explanation](../explanation/)** — understanding-oriented; the
  reader wants the *why* behind a design.
